I began to learn about computational complexity in MIT 600.1x, "Introduction to Computer Science and Programming Using Python". When I decided to tackle a first web development project, I wanted to review and solidify my understanding of this concept, how it works, and why it's useful. Since I found visualization helpful in this context, I decided to learn about Matplotlib, a Python 2D plotting library, and use it in a web page to present dynamic graphs that accept user input.

In searching for a starting point, I came across mpld3, an API for exporting Matplotlib figures (plots) to HTML. Then I discovered a demo project on GitHub that utilized this API combined with Flask, a microframework for Python web apps. Flask is accessible enough to begin using very quickly but also highly extensible, and thus appears to be a good platform for continuing to learn as I build more complex projects. I decided to fork the demo and started experimenting. (There is little resemblance between that original demo and this project but it was a good springboard)

Building this site is teaching me the basics of git workflow from the command line: checking status, diffing and staging changes, doing commits with messages, pushing to a remote repository. I'm also learning about deploying to a cloud PaaS, specifically Heroku. This site runs on a web dyno and I am pushing changes to my Heroku remote simultaneously. I am developing locally using the Heroku CLI inside a Python virtual environment to run a local server and observe changes immediately. It's a lot of fun and I can't wait to learn more!

At present this site uses Python, Flask, Matplotlib and mpld3 on the backend to generate figures (plots) of algorithm running time growth rates and export them into the site HTML. On the frontend it uses JavaScript for behaviour, interaction and Ajax requests. The source and a full list of resources used is in the GitHub repository.

Thanks for reading!
